1. 文档概述与目标

1.1 Phase 1 目标

核心目标：运行一个 Android "Hello World" 应用，验证 Android Framework 与 OpenHarmony 系统服务对接的软件框架可行性。

具体验证项：

  1. 适配层初始化 — App 启动时自动初始化适配层，连接到 OH 系统服务
  2. startActivity 拦截 — 拦截 startActivity(Intent)，转换为 OH StartAbility(Want)
  3. 生命周期映射 — Android Activity 生命周期与 OH Ability 生命周期双向映射
  4. 反向回调 — OH 系统服务的回调通过 JNI 回传到 Android 层
  5. 框架可扩展 — 建立可扩展的软件骨架，支持后续功能增量迭代
Stub 模式与真实 IPC：Phase 1 最初以 Stub 模式实现（日志打印模拟 IPC 调用），验证调用链路正确性。后续将所有 OH IPC 客户端升级为通过 OH SystemAbilityManager 获取真实代理对象，实现跨进程 IPC 调用。本文档涵盖从 Stub 到真实 IPC 的完整设计。

1.2 实现范围与验证指标

本文档从"运行一个 Hello World 应用"这一目标出发，完成以下工作：

  1. 最小接口集分析：分析 Android Hello World 应用的完整启动流程，识别必须适配的最小接口方法集合
  2. 四大任务设计与实现：C++ 真实 IPC、Java 反向回调、Surface 渲染桥接、输入事件桥接
  3. 端到端调用流程：从应用启动到首帧渲染的完整调用时序

3.2 ServiceInterceptor — 系统服务拦截器

属性说明
文件framework/core/java/ServiceInterceptor.java
职责通过 Java 动态代理拦截 Android 系统服务的 Binder 调用（作为 runtime 反射回退方案）
拦截方式反射替换 ActivityTaskManager.IActivityTaskManagerSingleton 的 mInstance

拦截机制：

原始调用链:
  startActivity() -> ActivityTaskManager.getService().startActivity(...)
                     ^ 返回 IActivityTaskManager.Stub.Proxy (连接 SystemServer)

拦截后:
  startActivity() -> ActivityTaskManager.getService().startActivity(...)
                     ^ 返回 Java Dynamic Proxy
                       -> ActivityTaskManagerHandler.invoke()
                         -> handleStartActivity()
                           -> IntentWantConverter.intentToWant(intent)
                           -> nativeStartAbility(wantParams...)
设计决策：AOSP 源码修改方案（通过 OHEnvironment.isOHEnvironment() 直接返回 Adapter 对象）是主要路径。ServiceInterceptor 作为 runtime 反射回退方案保留，用于未修改 AOSP 源码的场景。

4.1 OHAbilityManagerClient

属性说明

连接机制

// oh_ability_manager_client.cpp - connect()

bool OHAbilityManagerClient::connect() {
    auto samgr = OHOS::SystemAbilityManagerClient::GetInstance()
                     .GetSystemAbilityManager();
    if (samgr == nullptr) return false;

    auto remoteObject = samgr->GetSystemAbility(ABILITY_MGR_SERVICE_ID); // 180
    if (remoteObject == nullptr) return false;

    proxy_ = OHOS::iface_cast<OHOS::AAFwk::IAbilityManager>(remoteObject);
    return proxy_ != nullptr;
}

startAbility 实现

// oh_ability_manager_client.cpp - startAbility()

int OHAbilityManagerClient::startAbility(const WantParams& want) {
    if (proxy_ == nullptr) return -1;

    // Build OH Want object
    OHOS::AAFwk::Want ohWant;
    OHOS::AppExecFwk::ElementName element("", want.bundleName, want.abilityName);
    ohWant.SetElement(element);

    if (!want.action.empty())
        ohWant.SetAction(want.action);
    if (!want.uri.empty())
        ohWant.SetUri(want.uri);

    // Real OH IPC call
    int result = proxy_->StartAbility(ohWant);
    return result;
}

4.2 OHAppMgrClient

关键设计：AppSchedulerAdapter 注册

attachApplication() 是适配层与 OH AppMgr 建立通信的关键。 调用时，将本进程的 AppSchedulerAdapter（继承 OH AppSchedulerHost）注册到 OH AppMgr， OH 系统后续通过此 Stub 回调通知应用进程的生命周期事件。

// oh_app_mgr_client.cpp - attachApplication()

bool OHAppMgrClient::attachApplication(int pid, int uid,
                                        const std::string& processName) {
    if (proxy_ == nullptr) return false;

    // Create AppSchedulerAdapter (OH AppSchedulerHost subclass)
    // Holds JNI context, calls Java AppSchedulerBridge on OH callback
    appScheduler_ = new AppSchedulerAdapter(jvm_, appThread_, processName);

    // Register with OH AppMgr
    proxy_->AttachApplication(appScheduler_);
    return true;
}

4.3 OHWindowManagerClient

属性说明
文件framework/window/jni/oh_window_manager_client.h/cpp (新文件)
OH 目标服务ISceneSessionManager (SA 4607, 系统单例) — 会话创建/销毁
ISession (每窗口代理) — 会话操作 (由 CreateAndConnect 返回)
主要方法connect(), createSession(), updateSessionRect(), notifyDrawingCompleted(), destroySession()
关键架构：ISceneSessionManager vs ISession
ISceneSessionManager（SA 4607）是系统级单例，仅负责会话的创建（CreateAndConnectSpecificSession） 和销毁（DestroyAndDisconnectSpecificSession）。
ISession 是 CreateAndConnectSpecificSession 返回的每窗口代理， 后续所有窗口操作（UpdateSessionRect、DrawingCompleted、Foreground、Background、Disconnect 等） 均通过 ISession 进行，而非 ISceneSessionManager。
ISceneSessionManager 与 IWindowManager 的关系： ISceneSessionManager : public IWindowManager — 是 IWindowManager 的超集。 只需连接 SA 4607（ISceneSessionManager），无需单独适配 SA 4606（IWindowManager）。

OHWindowSession 数据结构

// oh_window_manager_client.h

struct OHWindowSession {
    int32_t sessionId = -1;        // OH window session ID
    int32_t surfaceNodeId = -1;    // OH RSSurfaceNode ID (for rendering)
    int32_t displayId = 0;         // Display ID
    int32_t width = 0;
    int32_t height = 0;
    bool valid = false;
};

// SessionEntry - internal session record holding ISession proxy
struct SessionEntry {
    int32_t sessionId;
    int64_t surfaceNodeId;
    sptr<ISession> sessionProxy;       // per-window ISession proxy (key!)
    sptr<SessionStageAdapter> stageAdapter;
    sptr<WindowCallbackAdapter> windowAdapter;
};

createSession 实现

// oh_window_manager_client.cpp - createSession()

OHWindowSession OHWindowManagerClient::createSession(...) {
    // Create reverse callback stubs
    auto stageAdapter = new SessionStageAdapter(jvm, androidWindow);
    auto windowAdapter = new WindowCallbackAdapter(jvm, androidWindow);

    // Create session via system singleton ISceneSessionManager
    auto ssmInterface = iface_cast<ISceneSessionManager>(ssmProxy_);
    sptr<ISession> session = nullptr;   // receives per-window proxy
    int32_t persistentId = 0;

    auto ret = ssmInterface->CreateAndConnectSpecificSession(
        stageAdapter, windowAdapter, sessionInfo,
        session,      // output: ISession per-window proxy
        token, windowProperty, persistentId);

    // Store ISession proxy in SessionEntry for later use
    sessions_[sessionId] = {sessionId, surfaceNodeId,
                            session,
                            stageAdapter, windowAdapter};
    return result;
}

每窗口操作 — 使用 ISession

// updateSessionRect - via ISession proxy
int OHWindowManagerClient::updateSessionRect(int32_t sessionId, ...) {
    auto sessionProxy = sessions_[sessionId].sessionProxy;  // ISession
    return sessionProxy->UpdateSessionRect(rect, reason);   // ISession method
}

// notifyDrawingCompleted - via ISession proxy
int OHWindowManagerClient::notifyDrawingCompleted(int32_t sessionId) {
    auto sessionProxy = sessions_[sessionId].sessionProxy;
    return sessionProxy->DrawingCompleted();
}

// destroySession - ISession.Disconnect() + SSM.DestroyAndDisconnect()
void OHWindowManagerClient::destroySession(int32_t sessionId) {
    sessionProxy->Disconnect();
    ssmInterface->DestroyAndDisconnectSpecificSession(id);
}

5. 反向回调实现（OH → Android）

核心机制：当 OH 系统服务通过 C++ Stub（如 AppSchedulerHost）回调应用进程时， C++ 层通过 JNI 调用 Java 层的 Bridge 类静态方法，Bridge 类构造 Android ClientTransaction 并交给 ActivityThread 执行，从而驱动 Android Activity 的生命周期。

这是 Hello World 运行最关键的方法。当 OH AbilityMgr 决定启动一个 Ability 时， 通过 AppSchedulerHost 回调此方法。该方法在 Java 侧构造完整的 Android Activity 启动事务。

// AppSchedulerBridge.java

public static void nativeOnScheduleLaunchAbility(
        String abilityName, String bundleName,
        int abilityType, int token) {

    // 1. Get ActivityThread and ApplicationThread
    ActivityThread activityThread = ActivityThread.currentActivityThread();
    IApplicationThread appThread = activityThread.getApplicationThread();

    // 2. Create Activity token
    IBinder activityToken = new Binder();

    // 3. OH abilityName -> Android className
    String className = IntentWantConverter.abilityNameToClassName(abilityName);
    String packageName = bundleName; // Phase 1: direct mapping

    // 4. Build Intent
    Intent intent = new Intent();
    intent.setComponent(new ComponentName(packageName, className));

    // 5. Build ActivityInfo
    ActivityInfo info = new ActivityInfo();
    info.packageName = packageName;
    info.name = className;
    info.applicationInfo = activityThread.getApplication().getApplicationInfo();

    // 6. Build ClientTransaction
    ClientTransaction transaction = ClientTransaction.obtain(appThread, activityToken);
    transaction.addCallback(LaunchActivityItem.obtain(
        intent, info, new Configuration(), null, null, 0, null, null));
    transaction.setLifecycleStateRequest(ResumeActivityItem.obtain(true));

    // 7. Register token mapping
    LifecycleAdapter.getInstance().registerActivityToken(token, activityToken);

    // 8. Dispatch -> triggers Activity.onCreate + onResume
    appThread.scheduleTransaction(transaction);
}
语义鸿沟：OH 的 Ability 启动是一个单步操作（ScheduleLaunchAbility）， 而 Android 的 Activity 启动需要构造包含 LaunchActivityItem + ResumeActivityItem 的 ClientTransaction。 Bridge 方法必须补全 Android 侧所需的 ActivityInfo、Configuration 等上下文。

// LifecycleAdapter.java - dispatchAndroidLifecycle()

private void dispatchAndroidLifecycle(int token, int androidState) {
    ActivityThread activityThread = ActivityThread.currentActivityThread();
    IApplicationThread appThread = activityThread.getApplicationThread();
    IBinder activityToken = getActivityToken(token);

    ClientTransaction transaction = ClientTransaction.obtain(appThread, activityToken);

    switch (androidState) {
        case ANDROID_STATE_RESUMED:
            transaction.setLifecycleStateRequest(
                ResumeActivityItem.obtain(true /* isForward */));
            break;
        case ANDROID_STATE_PAUSED:
            transaction.setLifecycleStateRequest(
                PauseActivityItem.obtain());
            break;
        case ANDROID_STATE_STOPPED:
            transaction.setLifecycleStateRequest(
                StopActivityItem.obtain(0 /* configChanges */));
            break;
        case ANDROID_STATE_DESTROYED:
            transaction.setLifecycleStateRequest(
                DestroyActivityItem.obtain(false, 0));
            break;
    }

    appThread.scheduleTransaction(transaction);
}

// WindowSessionAdapter.java - addToDisplay()

public int addToDisplay(IWindow window, WindowManager.LayoutParams attrs,
        int viewVisibility, int layerStackId, int requestedVisibleTypes,
        InputChannel outInputChannel, InsetsState insetsState,
        InsetsSourceControl.Array activeControls, Rect attachedFrame,
        float[] sizeCompatScale) throws RemoteException {

    String windowName = attrs.getTitle() != null
        ? attrs.getTitle().toString() : "AndroidWindow";
    int width = attrs.width > 0 ? attrs.width : 1080;
    int height = attrs.height > 0 ? attrs.height : 2340;

    // Create OH window session via native
    int[] sessionInfo = nativeCreateSession(
        window.asBinder(), windowName,
        attrs.type, layerStackId, width, height);
    // sessionInfo: [sessionId, surfaceNodeId, displayId, width, height]

    if (sessionInfo == null || sessionInfo[0] < 0) return -1;

    // Cache session info for relayout/finishDrawing/remove
    mSessionMap.put(window.asBinder(), sessionInfo);

    // Create InputChannel pair for input event delivery
    if (outInputChannel != null) {
        InputChannel clientChannel = InputEventBridge.getInstance()
            .createInputChannelPair(window.asBinder(), sessionInfo[0], windowName);
        clientChannel.transferTo(outInputChannel);
    }

    // Initialize output parameters
    if (insetsState != null) insetsState.set(new InsetsState());
    if (sizeCompatScale != null) sizeCompatScale[0] = 1.0f;

    return 0;
}

relayout() 实现

// WindowSessionAdapter.java - relayout()

public int relayout(IWindow window, WindowManager.LayoutParams attrs,
        int requestedWidth, int requestedHeight, int viewVisibility,
        int flags, int seq, int lastSyncSeqId, ClientWindowFrames outFrames,
        MergedConfiguration outMergedConfiguration, SurfaceControl outSurfaceControl,
        InsetsState insetsState, InsetsSourceControl.Array activeControls,
        Bundle bundle) throws RemoteException {

    int[] sessionInfo = mSessionMap.get(window.asBinder());
    if (sessionInfo == null) return -1;

    int sessionId = sessionInfo[0];
    int width = requestedWidth > 0 ? requestedWidth : sessionInfo[3];
    int height = requestedHeight > 0 ? requestedHeight : sessionInfo[4];

    // 1. Update OH session rect
    nativeUpdateSessionRect(sessionId, 0, 0, width, height);

    // 2. Fill output frame info
    if (outFrames != null) {
        Rect frame = new Rect(0, 0, width, height);
        outFrames.frame.set(frame);
        outFrames.displayFrame.set(frame);
        outFrames.parentFrame.set(frame);
    }

    // 3. Fill MergedConfiguration
    if (outMergedConfiguration != null) {
        Configuration config = new Configuration();
        config.screenWidthDp = width * 160 / 320;
        config.screenHeightDp = height * 160 / 320;
        config.densityDpi = 320;
        outMergedConfiguration.setOverrideConfiguration(config);
    }

    // 4. Create SurfaceControl (associated with OH RSSurfaceNode)
    if (outSurfaceControl != null) {
        long surfaceNodeId = nativeGetSurfaceNodeId(sessionId);
        SurfaceControl sc = new SurfaceControl.Builder()
            .setName("OH_Surface_" + sessionId)
            .setBufferSize(width, height)
            .build();
        outSurfaceControl.copyFrom(sc, "OH_relayout");
    }

    return 0;
}

7. 输入事件桥接

目标：建立 OH MMI → Android InputChannel 的输入事件管道，使触摸事件能从 OH 输入子系统正确传递到 Android View 层级，验证完整的输入处理流程。

7.1 输入事件管道架构

Android 的输入事件通过 InputChannel（基于 socketpair）从系统侧传递到应用侧。 在适配层中，没有 Android 的 InputDispatcher。取而代之的是 OH 的 MultiModal Input (MMI) 框架。 适配层需要：

  1. 在 addToDisplay() 时创建 Android InputChannel 对，client 端给 ViewRootImpl
  2. 保留 server 端，由 InputEventBridge 管理
  3. 当 OH MMI 向窗口投递触摸事件时，在 native 层转换为 Android InputMessage 格式
  4. 通过 InputPublisher（或直接写入 server fd）投递给 ViewRootImpl

public InputChannel createInputChannelPair(IBinder windowToken, int sessionId,
                                            String windowName) {
    String channelName = windowName + " (OH session " + sessionId + ")";
    InputChannel[] channels = InputChannel.openInputChannelPair(channelName);

    // channels[0] = server (adapter write end)
    // channels[1] = client (ViewRootImpl read end)
    InputChannel serverChannel = channels[0];
    InputChannel clientChannel = channels[1];

    mServerChannels.put(sessionId, serverChannel);
    mWindowSessionMap.put(windowToken, sessionId);

    // Register server channel fd to native InputPublisher
    nativeRegisterInputChannel(sessionId, serverChannel);

    return clientChannel;
}

// oh_input_bridge.cpp - writeMotionEvent()

int OHInputBridge::writeMotionEvent(int fd, uint32_t seq, int32_t action,
                                     float x, float y,
                                     int64_t downTime, int64_t eventTime) {
    struct {
        InputMessageHeader header;
        MotionEventBody body;
    } msg;
    memset(&msg, 0, sizeof(msg));

    msg.header.type = INPUT_MSG_TYPE_MOTION; // 2
    msg.header.seq = seq;

    msg.body.source = AINPUT_SOURCE_TOUCHSCREEN;
    msg.body.action = action;
    msg.body.downTime = downTime;
    msg.body.eventTime = eventTime;

    // Single finger touch
    msg.body.pointerCount = 1;
    msg.body.pointerProperties.id = 0;
    msg.body.pointerProperties.toolType = AMOTION_EVENT_TOOL_TYPE_FINGER;
    msg.body.pointerCoords.bits = AXIS_X_BIT | AXIS_Y_BIT | AXIS_PRESSURE_BIT;
    msg.body.pointerCoords.values[0] = x;     // X
    msg.body.pointerCoords.values[1] = y;     // Y
    msg.body.pointerCoords.values[2] = 1.0f;  // PRESSURE

    // Identity transform
    msg.body.dsdx = 1.0f;
    msg.body.dsdy = 1.0f;

    ssize_t written = send(fd, &msg, sizeof(msg), MSG_DONTWAIT | MSG_NOSIGNAL);
    return (written > 0) ? 0 : -errno;
}
