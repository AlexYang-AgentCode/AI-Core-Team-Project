一、第一阶段目标

核心目标：运行一个 Android "Hello World" 应用，验证 Android Framework 与 OpenHarmony 系统服务对接的软件框架可行性。

具体验证项：

  1. 适配层初始化 — App 启动时自动初始化适配层，连接到 OH 系统服务
  2. startActivity 拦截 — 拦截 startActivity(Intent)，转换为 OH StartAbility(Want)
  3. 生命周期映射 — Android Activity 生命周期与 OH Ability 生命周期双向映射
  4. 反向回调 — OH 系统服务的回调通过 JNI 回传到 Android 层
  5. 框架可扩展 — 建立可扩展的软件骨架，支持后续功能增量迭代
第一阶段采用 Stub 模式：OH IPC 客户端以日志打印模拟真实的 IPC 调用，验证整个调用链路的正确性。后续阶段替换为真实的 OH IPC 调用。

四、模块设计

4.1 OHEnvironment — 适配层主入口

属性说明
文件framework/core/java/OHEnvironment.java
职责单例模式，管理适配层生命周期、持有各子模块引用、声明 JNI native 方法
初始化时机Application.onCreate()

初始化流程：

  1. nativeInitialize() — 初始化 JNI Bridge，缓存 Java 方法 ID
  2. nativeConnectToOHServices() — 通过 OH SystemAbilityManager 获取远程代理
  3. 创建 LifecycleAdapter 实例
  4. 创建 ServiceInterceptor 并安装拦截器
  5. nativeAttachApplication() — 向 OH AppMgr 注册当前进程

JNI Native 方法清单：

方法方向对应 OH 接口
nativeInitialize()初始化IPC 框架初始化
nativeConnectToOHServices()初始化SystemAbilityManager.GetSystemAbility()
nativeAttachApplication(pid, uid, pkg)App→OHIAppMgr.AttachApplication()
nativeStartAbility(bundle, ability, ...)App→OHIAbilityManager.StartAbility(Want)
nativeConnectAbility(bundle, ability, cb)App→OHIAbilityManager.ConnectAbility(Want)
nativeDisconnectAbility(connId)App→OHIAbilityManager.DisconnectAbility()
nativeNotifyAppState(state)App→OHIAppMgr.ApplicationForegrounded/Backgrounded()
nativeShutdown()清理断开连接，释放资源

4.2 ServiceInterceptor — 系统服务拦截器

属性说明
文件framework/core/java/ServiceInterceptor.java
职责通过 Java 动态代理拦截 Android 系统服务的 Binder 调用
拦截方式反射替换 ActivityTaskManager.IActivityTaskManagerSingleton 的 mInstance

拦截机制：

原始调用链:
  startActivity() → ActivityTaskManager.getService().startActivity(...)
                     ↑ 返回 IActivityTaskManager.Stub.Proxy (连接 SystemServer)

拦截后:
  startActivity() → ActivityTaskManager.getService().startActivity(...)
                     ↑ 返回 Java Dynamic Proxy
                       → ActivityTaskManagerHandler.invoke()
                         → handleStartActivity()
                           → IntentWantConverter.intentToWant(intent)
                           → nativeStartAbility(wantParams...)
设计决策：选择 Java 动态代理而非修改 AIDL 生成代码，是因为动态代理不需要修改 Android Framework 的 AIDL 文件，降低了侵入性，便于维护和升级。

4.3 IntentWantConverter — Intent/Want 转换器

属性说明
文件framework/activity/java/IntentWantConverter.java
职责Android Intent 与 OH Want 之间的双向转换

映射规则：

Android Intent 字段OH Want 字段转换规则
component.packageNamebundleName第一阶段直接映射，后续查表
component.classNameabilityName取 SimpleName + "Ability" 后缀
actionactionAction 映射表（MAIN→home, VIEW→viewData 等）
data (URI)uri直接传递
extras (Bundle)parametersBundle 序列化为 JSON

4.4 LifecycleAdapter — 生命周期适配器

属性说明
文件framework/activity/java/LifecycleAdapter.java
职责Android Activity 与 OH Ability 生命周期状态的双向映射

状态映射表：

Android StateOH State说明
CREATED / STARTEDINACTIVE (1)已创建但未进入前台
RESUMEDFOREGROUND (2)前台可交互
PAUSEDFOREGROUND (2)OH 无 pause 对应，保持前台
STOPPEDBACKGROUND (4)后台
DESTROYEDINITIAL (0)已销毁

4.5 JNI Bridge — Java/C++ 桥接层

属性说明
文件framework/core/jni/adapter_bridge.h/cpp
职责实现 JNI native 方法，管理 JVM 引用和 Java 回调
共享库名liboh_adapter_bridge.so

关键设计：

  • JNI_OnLoad 保存 JavaVM 指针，供 OH 回调线程使用
  • AdapterBridge 单例管理 Global Ref 和缓存的 jmethodID
  • OH 回调线程通过 AttachCurrentThread 获取 JNIEnv

4.6 OH IPC 客户端

文件对应 OH 接口说明
oh_ability_manager_client.h/cppIAbilityManagerStartAbility、ConnectAbility、DisconnectAbility
oh_app_mgr_client.h/cppIAppMgrAttachApplication、状态通知

第一阶段以 Stub 模式实现（日志打印 + 模拟返回值），代码中以注释标注了真实实现的完整代码。

4.7 OH 回调处理器

属性说明
文件framework/core/jni/oh_callback_handler.h/cpp
职责实现 OH IRemoteStub 接口（AppSchedulerHost、AbilitySchedulerStub），接收 OH 系统服务回调
回调路径OH OnRemoteRequest → C++ Handler → JNI → Java LifecycleAdapter.onOHLifecycleCallback()

七、构建与运行

7.1 AOSP 集成构建

# 将 adapter 目录放入 AOSP 源码树
cp -r D:/code/adapter/ $AOSP_ROOT/packages/apps/OHAdapter/

# 构建
cd $AOSP_ROOT
source build/envsetup.sh
lunch aosp_arm64-userdebug
m OHAdapterHelloWorld

7.2 独立 NDK 构建（JNI 库）

cd D:/code/adapter/framework/jni
mkdir build && cd build
cmake -DCMAKE_TOOLCHAIN_FILE=$NDK/build/cmake/android.toolchain.cmake \
      -DANDROID_ABI=arm64-v8a \
      -DANDROID_PLATFORM=android-30 \
      ..
make -j8

7.3 验证方法

# 安装并运行
adb install OHAdapterHelloWorld.apk
adb shell am start -n com.example.helloworld/.MainActivity

# 查看适配层日志
adb logcat -s OH_Adapter OH_JNI_Bridge OH_AbilityMgrClient OH_AppMgrClient OH_CallbackHandler OH_LifecycleAdapter OH_ServiceInterceptor OH_IntentWantConverter HelloWorld
