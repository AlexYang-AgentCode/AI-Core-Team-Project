一、环境总览与架构

本项目（Android-OpenHarmony 适配层）需要修改 OH 系统服务源码，因此必须搭建完整的 OH 系统级开发环境。采用"华为云远程编译 + 本地 PC 调试"的分离架构：

┌─────────────────────────────────────────────────────────────────────────────┐ │ 整体环境架构 │ │ │ │ ┌──────────────────────┐ SSH/rsync ┌────────────────────────┐ │ │ │ 本地 PC (Windows) │ ◄═══════════════► │ 华为云 ECS (Ubuntu) │ │ │ │ │ 互联网 │ │ │ │ │ · 源码编辑 │ │ · OH 全量源码 │ │ │ │ · adapter 开发 │ ① 上传 patch │ · 编译工具链 │ │ │ │ · hdc 设备调试 │ ─────────────► │ · 增量/全量编译 │ │ │ │ · 日志查看 │ │ │ │ │ │ │ ② 下载 .so 产物 │ 编译产出: │ │ │ │ WSL2 Ubuntu │ ◄───────────── │ · libabilityms.z.so │ │ │ │ · rsync 同步 │ │ · libwms.z.so │ │ │ │ · 自动化脚本 │ │ · 系统镜像 │ │ │ └──────────┬───────────┘ └────────────────────────┘ │ │ │ USB │ │ │ hdc │ │ ┌──────────▼───────────┐ │ │ │ DAYU600 开发板 │ ③ hdc file send .so │ │ │ (OpenHarmony 系统) │ ④ kill foundation → 自动重启 │ │ │ RK3588 / 8核 │ ⑤ hdc hilog 查看日志 │ │ │ · foundation 进程 │ │ │ │ (AMS/WMS/...) │ │ │ └──────────────────────┘ │ └─────────────────────────────────────────────────────────────────────────────┘

各节点职责明确：

RK3588 vs RK3568 对比：
  • CPU：RK3588 为 8 核（4×A76 + 4×A55），RK3568 为 4 核（4×A55）。RK3588 运算能力约为 RK3568 的 3.5 倍
  • GPU：RK3588 搭载 Mali-G610，性能约为 RK3568（Mali-G52）的 6 倍
  • NPU：RK3588 内置 6 TOPS AI 算力，RK3568 为 1 TOPS
  • 视频：RK3588 支持 8K 编解码，RK3568 仅支持 4K
  • 内存：RK3588 支持 LPDDR4X/LPDDR5，RK3568 仅支持 LPDDR4/DDR4

购买渠道：淘宝/京东搜索 "DAYU600 开发板" 或 "润开鸿 HH-SCDAYU600"。

不推荐华为手机：HarmonyOS NEXT 虽基于 OpenHarmony 内核，但华为做了大量闭源定制，系统分区受 AVB 验证保护，目前没有公开的 root / 解锁 bootloader 方案。不适合系统服务级开发。

3.4 SSH 密钥对配置

步骤 4：创建并下载密钥对

方式 A：在华为云控制台创建（推荐新手）：

  1. 控制台 → ECS → "密钥对" → "创建密钥对"
  2. 输入名称（如 oh-dev-key）
  3. 点击创建，浏览器会自动下载 .pem 私钥文件
  4. 将私钥文件保存到本地 PC：C:\Users\iceberg\.ssh\oh-dev-key.pem

方式 B：使用本地已有密钥：

# 在 Git Bash 或 WSL2 中生成密钥对
ssh-keygen -t ed25519 -C "oh-dev" -f ~/.ssh/oh-dev-key

# 然后将公钥内容复制到华为云控制台
cat ~/.ssh/oh-dev-key.pub
# 控制台 → 密钥对 → 导入密钥对 → 粘贴公钥内容

3.5 首次登录验证

步骤 5：SSH 连接测试

# 在 Git Bash 或 WSL2 终端中执行

# 设置私钥文件权限（必须，否则 SSH 拒绝使用）
chmod 600 ~/.ssh/oh-dev-key.pem

# 连接服务器（替换为你的 ECS 公网 IP）
ssh -i ~/.ssh/oh-dev-key.pem root@<ECS公网IP>

# 验证成功后，建议配置 SSH config 简化后续操作

在 ~/.ssh/config 中添加以下配置，后续可直接 ssh oh-build 登录：

Host oh-build
    HostName <ECS公网IP>
    User root
    Port 22
    IdentityFile ~/.ssh/oh-dev-key.pem   # 密钥登录时需要；密码登录可删除此行
    ServerAliveInterval 30               # 每 30 秒发心跳，防止连接被中间设备/安全组关闭
    ServerAliveCountMax 10               # 连续 10 次无响应才断开（即 5 分钟超时）
SSH 连接频繁断开问题：华为云 ECS 的安全组或中间网络设备会关闭空闲的 TCP 连接。ServerAliveInterval 必须配置，否则 SSH 会话在空闲几分钟后被强制断开。建议设为 30 秒。如果仍有断连问题，可进一步检查华为云安全组的入方向规则是否放通了你当前的公网 IP。

配置完成后验证：

# 简化登录
ssh oh-build

# 查看系统信息
uname -a
df -h
free -h
nproc

4.3 下载 OH 系统源码

耗时提醒：OH 全量源码约 30-50 GB，在华为云服务器上下载预计 1-2 小时。
重要：必须先 cd 到目标目录再执行 repo init。repo init 和 repo sync 会在当前目录下创建 .repo/ 并 checkout 源码。如果在 /root/ 下执行，源码会散落在 home 目录中，与系统文件混在一起，难以管理。 
# 安装 screen（防止 SSH 断连导致下载中断）
apt-get install -y screen
screen -S oh-build

# ⚠ 必须先创建并进入目标目录
mkdir -p ~/openharmony && cd ~/openharmony

# 初始化 repo
repo init -u https://gitcode.com/openharmony/manifest \
    -b OpenHarmony-6.1-Release --no-repo-verify

# 开始同步源码（使用 8 线程加速）
repo sync -c -j8 --force-sync
# 如果中途 SSH 断连，screen 会保护进程继续运行
# 按 Ctrl+A 再按 D 可安全脱离 screen
# 用 screen -r oh-build 重新进入查看进度

# 同步完成后查看大小
du -sh ~/openharmony/
版本选择建议：
  • OpenHarmony-6.1-Release — 最新稳定版本（2026年3月发布），推荐首选
  • OpenHarmony-5.0-Release — 上一代版本，较为成熟
  • 版本必须与 DAYU600 烧录的镜像版本一致
  • OH 源码主托管地址已切换至 gitcode.com（原 gitee.com 不再是主托管）

4.5 首次全量编译

首次全量编译耗时约 1.5-3 小时（16核60G），请确保服务器不会被意外关机。建议使用 screen 在后台执行，防止 SSH 断连导致编译中断。
Ubuntu 24.04 兼容性修复：OH 6.1 编译系统默认仅支持 Ubuntu 18.04/20.04/22.04，在 24.04 上会报 "OS version 24.04 is not supported for ohos build"。需先手动添加 24.04 支持： 
# 将 24.04 加入支持列表
sed -i "s/('18.04', '20.04', '22.04')/('18.04', '20.04', '22.04', '24.04')/" \
    ~/openharmony/build/scripts/tools_checker.py
产品名称说明：DAYU600 (RK3588) 在 OH 6.1 源码中对应的产品名称为 dayu210（位于 vendor/hihope/dayu210/），而非 rk3588。使用错误的产品名会报 "product rk3588@None not found"。
SDK 组件预编译：OH 6.1 全量编译前需要先构建 SDK 组件，否则 GN 阶段会报 "OHOS component : (sdk) not found"。需先执行： 
# 先构建 SDK
./build.sh --product-name dayu210 --ccache --build-target ohos_sdk 2>&1 | tee build_sdk.log
SDK 构建完成后再执行全量编译。 
# 在 screen 会话中执行（screen -r oh-build 或 screen -S oh-build）
cd ~/openharmony

# 第一步：构建 SDK 组件（首次编译必须）
./build.sh --product-name dayu210 --ccache --build-target ohos_sdk 2>&1 | tee build_sdk.log

# 第二步：全量编译
./build.sh --product-name dayu210 --ccache 2>&1 | tee build_full.log

# 编译完成后查看结果
echo $?  # 0 表示成功

# 查看编译产物（镜像文件）
ls -lh out/rk3588/packages/phone/images/

# 退出 screen (保持后台运行): 按 Ctrl+A, 然后按 D
# 重新进入 screen: screen -r oh-build

6.3.1 进入烧录模式

  1. 断开电源
  2. 用 Type-C 数据线连接开发板 OTG 口到 PC
  3. 按住 MaskROM 按键，接通电源
  4. RKDevTool 显示 "发现一个 MASKROM 设备" 后，松开按键

6.3.2 方式 A：整包烧录（推荐，首次刷机必选）

7.1 同步方案概述

脚本 1：remote_config.sh — 配置文件

#!/bin/bash
# ===== 远程服务器配置（使用前必须修改） =====
REMOTE_HOST="xxx.xxx.xxx.xxx"      # 华为云 ECS 公网 IP
REMOTE_USER="root"
REMOTE_PORT=22
SSH_KEY="~/.ssh/oh-dev-key.pem"   # SSH 私钥路径

# ===== 远程路径 =====
REMOTE_OH_ROOT="~/openharmony"      # 远程 OH 源码根目录
REMOTE_BUILD_OUT="${REMOTE_OH_ROOT}/out/rk3588"

# ===== 本地路径（WSL2 下的路径） =====
LOCAL_ADAPTER_DIR="/mnt/d/code/adapter"
LOCAL_OHOS_SRC="/mnt/d/code/ohos"
LOCAL_BUILD_CACHE="/mnt/d/code/adapter/build/remote_artifacts"

# ===== 编译配置 =====

脚本 2：sync_upload.sh — 上传 patch 到远程服务器

#!/bin/bash
# 将本地 ohos_patches 上传到远程服务器并 apply
source "$(dirname "$0")/remote_config.sh"

echo "========== [上传] OH Patch 文件 =========="

# 同步 ohos_patches 到远程暂存目录
rsync -avz --progress --delete \
    -e "${RSYNC_SSH}" \
    "${LOCAL_ADAPTER_DIR}/ohos_patches/" \
    "${REMOTE_USER}@${REMOTE_HOST}:${REMOTE_OH_ROOT}/_adapter_patches/"

echo "========== [远程] Apply Patches =========="

${SSH_CMD} bash -s <<'REMOTE_SCRIPT'
cd ~/openharmony
PATCH_DIR="./_adapter_patches"

# 遍历并覆盖源文件（非 .patch 格式的直接覆盖）
find "${PATCH_DIR}/ability_rt" -type f \( -name "*.cpp" -o -name "*.h" \) 2>/dev/null | while read sf; do
    rel_path=$(echo "$sf" | sed 's|.*_adapter_patches/ability_rt/||')
    target="foundation/ability/ability_runtime/${rel_path}"
    if [ -d "$(dirname "$target")" ]; then
        echo "  覆盖: $target"
        cp "$sf" "$target"
    else
        echo "  跳过（目标目录不存在）: $target"
    fi
done

# 应用 .patch 格式的补丁文件
find "${PATCH_DIR}" -name "*.patch" -type f 2>/dev/null | while read pf; do
    echo "  Apply patch: $pf"
    git apply "$pf" 2>/dev/null && echo "    成功" || echo "    已应用或冲突"
done

echo "Done."
REMOTE_SCRIPT

echo "========== 上传完成 =========="

脚本 3：remote_build.sh — 远程增量编译

#!/bin/bash

脚本 4：sync_download.sh — 下载编译产物

#!/bin/bash
# 从远程服务器下载编译产物到本地

脚本 5：deploy_device.sh — 部署到开发板

#!/bin/bash
# 将编译产物通过 hdc 推送到 OH 开发板
source "$(dirname "$0")/remote_config.sh"

ARTIFACT_DIR="${LOCAL_BUILD_CACHE}"

echo "========== [检查] 设备连接 =========="
DEVICE=$(hdc list targets 2>/dev/null | head -1)
if [ -z "$DEVICE" ] || [ "$DEVICE" = "[Empty]" ]; then
    echo "错误：未检测到 OH 设备，请检查 USB 连接"
    exit 1
fi
echo "设备: $DEVICE"

echo ""
echo "========== [挂载] 系统分区为可写 =========="
hdc shell mount -o remount,rw /

echo ""
echo "========== [推送] 文件到设备 =========="

for so_file in "${ARTIFACT_DIR}"/*.so; do
    [ -f "$so_file" ] || continue
    filename=$(basename "$so_file")
    echo "  推送: ${filename} -> /system/lib64/"
    hdc file send "$so_file" "/system/lib64/${filename}"
done

echo ""
echo "========== [重启] foundation 服务 =========="
FPID=$(hdc shell pidof foundation 2>/dev/null)
if [ -n "$FPID" ]; then
    echo "  终止 foundation (PID: $FPID)"
    hdc shell kill -9 $FPID
    sleep 3
fi

# 验证 foundation 重新启动
NEW_FPID=$(hdc shell pidof foundation 2>/dev/null)
if [ -n "$NEW_FPID" ]; then
    echo "  foundation 已重新启动 (PID: $NEW_FPID)"
else
    echo "  警告：foundation 未自动重启，执行完全重启..."
    hdc shell reboot
fi

echo ""
echo "========== 部署完成 =========="

十一、注意事项与常见问题

11.1 版本一致性

最关键的一点：设备镜像版本、OH 源码版本、SDK 版本三者必须严格匹配。版本不一致会导致 .so 文件因 ABI 不兼容而 crash。
组件版本要求如何确认
设备烧录镜像必须完全相同
如 OpenHarmony-6.1-Release		hdc shell param get const.ohos.fullname
源码分支cd ~/openharmony && repo info
SDK 版本DevEco Studio 中查看

11.2 SELinux / 访问控制

# 查看 SELinux 状态
hdc shell getenforce

# 临时关闭 SELinux（调试用，重启后恢复）
hdc shell setenforce 0

# 如果推送的 .so 无法加载，检查文件权限
hdc shell ls -lZ /system/lib64/libabilityms.z.so
# 确保权限为 644，SELinux 上下文正确
hdc shell chmod 644 /system/lib64/libabilityms.z.so

11.3 华为云服务器省钱技巧

  • 不用时关机：控制台 → ECS → 关机（按量付费关机后 CPU/内存不计费，仅磁盘计费）
  • 不要释放实例：关机 ≠ 释放。释放会删除所有数据（包括已编译的源码）
  • 编译完立即关机：增量编译通常 5-10 分钟完成，关机后成本仅为磁盘费用
  • 善用 ccache：首次全量编译后，后续增量编译时间大幅缩短
  • 按需升降配：华为云支持在线变更 ECS 规格。日常可用 8核16G，全量编译时临时升级到 16核32G

11.4 常见问题排查

问题可能原因解决方案

apt-get upgrade 提示 sshd_config 修改 系统升级想覆盖 SSH 配置 选择 "keep the local version"（按 Enter 或输入 N），保留当前配置
