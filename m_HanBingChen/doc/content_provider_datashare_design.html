一、适配范围与需求

1.1 需要支持的场景

编号场景数据流方向说明
S1 Android App 之间的数据访问 Android App A → Android App B 的 ContentProvider App B 通过 publishContentProviders() 注册自身 CP，App A 通过 ContentResolver 访问。在适配层内部通过 ContentProviderRegistry 管理，不涉及 OH 系统服务。
S2 Android App 访问 OH 系统应用数据 Android App → OH DataShareExtensionAbility Android App 通过 ContentResolver 访问通讯录、短信、图库等系统数据，适配层将 content:// URI 转换为 datashare:// URI，通过 OH DataShareHelper (Stage 模型) 完成数据操作。

1.2 不需要支持的场景

场景原因
OH 原生应用访问 Android App 的 ContentProvider 不在项目适配范围内，无此需求
FA 模型 (Page Ability) 的 DataAbility 项目统一适配到 Stage 模型

1.3 适配目标模型

目标：Stage 模型的 DataShareExtensionAbility
OH 客户端 API：DataShareHelper（Stage 模型）
不使用 FA 模型的 DataAbilityHelper。

5.2 关键设计决策

决策方案原因
ContentProviderBridge 的基类 extends ContentProviderNative ContentProviderNative 是 IContentProvider 的 Binder Stub 实现。继承它可以作为合法的 IContentProvider 返回给 ContentResolver，走标准 Binder IPC 路径。
App 间 CP 管理 ContentProviderRegistry 单例 替代 AMS 中的 ContentProviderHelper，在适配层内部管理已发布的 Android CP。不涉及 OH 系统服务。
ContentValues 传递方式 拆为 keys[]/values[]/types[] 三个并行数组 避免在 JNI 层遍历 Java Map 的复杂反射。Java 层负责拆包，C++ 层按类型重建 ValuesBucket。
ResultSet→Cursor 转换 C++ 层使用 JNI 构建 MatrixCursor MatrixCursor 是 Android 的轻量级内存 Cursor 实现，适合中小结果集。避免了 CursorWindow 共享内存的复杂转换。
SQL Selection 解析 正则表达式简单解析 + SetWhereClause 降级 覆盖 90%+ 的常见 selection 模式。无法解析时降级为 OH 支持的 SetWhereClause 原始 SQL。
OH DataShareHelper 连接模型 首次 getContentProvider 时建立，缓存复用 OH DataShareHelper::Creator 是异步连接。在 nativeConnect 中同步等待连接完成，后续操作复用已建立的连接。

6.2 Java 层：ContentProviderRegistry

文件：framework/contentprovider/java/ContentProviderRegistry.java

单例，维护两个 Map：

  • mLocalProviders (ConcurrentHashMap<authority, ContentProviderHolder>) — App 自身发布的 CP

acquireProvider() 先查 mLocalProviders（场景 S1），未命中则创建/返回 ContentProviderBridge（场景 S2）。

6.4 C++ 层：OHDataShareClient

文件：framework/contentprovider/jni/oh_datashare_client.h/cpp

核心功能：

SQL Selection 解析器 (buildPredicates)

将 Android SQL 风格的 selection 字符串解析为 OH DataSharePredicates 链式调用：

SQL 操作符OH Predicates 方法
=EqualTo(column, value)
!= / <>NotEqualTo(column, value)
>GreaterThan(column, value)
<LessThan(column, value)
>=GreaterThanOrEqualTo(column, value)
<=LessThanOrEqualTo(column, value)
LIKELike(column, value)
IS NULLIsNull(column)
IS NOT NULLIsNotNull(column)
ANDAnd()
OROr()

无法解析的复杂 SQL 降级为 SetWhereClause(selection) + SetWhereArgs(selArgs)。

ResultSet → MatrixCursor 转换 (resultSetToCursor)

通过 JNI 调用构建 Java MatrixCursor：

  1. 从 DataShareResultSet 获取列名列表 → 创建 Java String[] → new MatrixCursor(columns)
  2. 遍历 ResultSet 每行：cursor.newRow() → 根据列类型(STRING/INTEGER/FLOAT/NULL)调用 rowBuilder.add(value)
  3. JNI 类/方法 ID 在 JNI_OnLoad 时缓存，避免每次查询重复查找
