## 🎯 5 分钟快速配置

### 前置准备
- ✅ 群晖 NAS 已开启 SSH 服务
- ✅ 知道 NAS 的 IP 或域名（如 `nas.local`）
- ✅ 有 NAS 管理员账号（通常是 root）

### 一键初始化

```bash
# 1. 进入 vault 根目录
cd /path/to/obsidian/vault

# 2. 下载并执行初始化脚本
bash 10-Projects/16-DigitalEmployee/Git-Hooks/本地初始化脚本.sh

# 3. 根据提示输入选项，完成！
```

**预期结果**：
```
✅ vault 初始化完成
✅ 已连接到群晖 Git Server
✅ 首次推送成功
```

## 🔄 日常工作流

### 推送修改到群晖

```bash
# 在 vault 根目录执行

# 1. 暂存所有更改
git add .

# 2. 提交
git commit -m "描述你的修改"

# 3. 推送（自动触发 NAS 上的 post-receive hook）
git push synology main

# 💡 可以设置 git 快捷命令：
# git config alias.pushn '!git add . && git commit -m "Quick save" && git push synology main'
# 后续只需: git pushn
```

### 定时自动备份（可选）

编辑 crontab：
```bash
crontab -e
```

添加以下行（每天 23:00 自动备份）：
```cron
0 23 * * * cd /path/to/vault && git add . && git commit -m "Daily auto backup: $(date '+\%Y-\%m-\%d')" && git push synology main 2>/dev/null || true
```

## ❌ 常见问题

### Q1: "Permission denied" 错误
```
git push synology main
Permission denied (publickey,gssapi-keyex,gssapi-with-mic)
```

**解决**:
```bash
# 1. 确保 NAS 的 SSH 服务已启用
# 群晖 → 控制面板 → 终端机和 SNMP → 启用 SSH 功能

# 2. 测试 SSH 连接
ssh -v root@nas.local

# 3. 如果仍失败，检查 SSH 密钥
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519
cat ~/.ssh/id_ed25519.pub | ssh root@nas.local "cat >> ~/.ssh/authorized_keys"
```

### Q2: "NAS 地址无法解析"
```
ssh: Could not resolve hostname nas.local: Name or service not known
```

**解决**:
- 用 IP 地址替代域名：`192.168.1.100`
- 编辑 `~/.ssh/config`：
  ```
  Host nas
      HostName 192.168.1.100
      User root
  ```
  然后用 `ssh nas` 连接

### Q3: 推送很慢或经常超时
```bash
# 增加 SSH 超时时间
git config --global ssh.ConnectTimeout 30

# 或在 ~/.ssh/config 中：
Host nas.local
    ConnectTimeout 30
    ServerAliveInterval 60
```

### Q4: 需要回退某个提交
```bash
# 查看历史
git log --oneline -10

# 回退到某个提交（不删除本地文件）
git reset --soft abc123def456

# 或彻底回退（删除修改）
git reset --hard abc123def456

# 强制推送（谨慎！）
git push -f synology main
```
