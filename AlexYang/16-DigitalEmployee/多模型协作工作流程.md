# 多模型协作工作流程 - 操作指导手册

> **核心原则**: Claude Code 用于高价值任务，廉价模型用于大量重复任务
> **更新时间**: 2026-03-11

---

## 📊 资源分层策略

### 模型能力与成本矩阵

| 模型 | Token成本 | 限制 | 优势 | 适用任务 |
|------|---------|------|------|---------|
| **Claude Code** | 高 | 5小时/100刀 | 推理强、代码质量高 | 架构设计、复杂Bug、关键决策 |
| **GLM (OpenCode)** | 低 | 无明显限制 | 中文好、便宜 | 文档生成、代码重构、测试用例 |
| **Kimi** | 低 | 无明显限制 | 长上下文、便宜 | 长文档分析、需求理解、知识检索 |
| **DeepSeek** | 极低 | 无明显限制 | 编码能力强 | 代码生成、代码审查、单元测试 |
| **OpenClaw (本地)** | 无 | 本地算力 | 无限制、隐私 | 敏感数据处理、批量文本分析 |

### 2. Git Hook 配置

文件: `Git-Hooks/post-commit.sh`

```bash
#!/bin/bash

# 检测是否有 Agent 状态文件变更
if git diff --cached --name-only | grep -q "AGENT-STATUS.md"; then
    echo "✅ Agent 状态已更新"
    
    # 提取状态信息
    STATUS=$(grep "^status:" AGENT-STATUS.md | awk '{print $2}')
    PROJECT=$(grep "^project:" AGENT-STATUS.md | awk '{print $2}')
    
    # 发送到 Obsidian (通过文件系统)
    echo "$(date): $PROJECT - $STATUS" >> /mnt/d/ObsidianVault/40-Journal/$(date +%Y-%m-%d).md
fi

# 检查是否需要切换模型
if git diff --cached --name-only | grep -q "architecture\|design\|决策"; then
    echo "⚠️ 检测到架构变更，建议使用 Claude Code 进行审查"
    echo "运行: claude-code"
fi

# 自动选择合适的模型继续工作
CHANGED_FILES=$(git diff --cached --name-only)
CODE_COUNT=$(echo "$CHANGED_FILES" | grep -c "\.py$\|\.js$\|\.ts$" || true)
DOC_COUNT=$(echo "$CHANGED_FILES" | grep -c "\.md$" || true)

if [ $CODE_COUNT -gt 3 ]; then
    echo "💡 检测到大量代码变更，建议使用 DeepSeek 继续开发"
    echo "运行: opencode (DeepSeek 已配置为默认)"
elif [ $DOC_COUNT -gt 2 ]; then
    echo "💡 检测到大量文档变更，建议使用 Kimi 继续文档工作"
fi
```

### 3. 自动化脚本

文件: `Git-Hooks/agent-orchestrator.sh`

```bash
#!/bin/bash

# Agent 编排器 - 自动选择最佳模型

TASK_TYPE=$1
INPUT_FILE=$2

case $TASK_TYPE in
    "design")
        echo "🎨 使用 Claude Code 进行架构设计..."
        claude-code "基于 $INPUT_FILE 设计系统架构"
        ;;
    "code")
        echo "💻 使用 DeepSeek 生成代码..."
        opencode "基于 $INPUT_FILE 实现代码"
        ;;
    "test")
        echo "🧪 使用 GLM 生成测试..."
        opencode --model glm-4 "为 $INPUT_FILE 生成测试用例"
        ;;
    "doc")
        echo "📝 使用 Kimi 生成文档..."
        opencode --model kimi-k2 "为 $INPUT_FILE 生成文档"
        ;;
    "review")
        echo "🔍 使用 Claude Code 进行代码审查..."
        claude-code "审查 $INPUT_FILE 的代码质量"
        ;;
    *)
        echo "未知任务类型: $TASK_TYPE"
        echo "支持: design, code, test, doc, review"
        exit 1
        ;;
esac
```

## 🎯 模型切换策略

### 自动切换规则

| 场景 | 推荐模型 | 切换条件 |
|------|---------|---------|
| 开始新功能开发 | DeepSeek | 默认 |
| 遇到复杂架构问题 | Claude Code | DeepSeek 失败 2 次 |
| 需要理解长文档 | Kimi | 文档 > 5000 字 |
| 批量测试生成 | GLM | 测试文件 > 3 个 |
| 代码审查 | Claude Code | 关键模块 |
| 常规代码审查 | DeepSeek | 非关键模块 |
| 敏感数据处理 | OpenClaw | 包含密码/密钥 |

### 手动切换命令

```bash
# 切换到 Claude Code (高价值任务)
claude-code

# 切换到 OpenCode + 指定模型
opencode --model deepseek-coder  # 编码
opencode --model kimi-k2         # 文档/分析
opencode --model glm-4           # 测试/重构

# 使用本地模型 (无限制)
openclaw --task "批量处理敏感配置文件"
```

## 💡 最佳实践

### 1. Claude Code 使用原则

✅ **应该用:**
- 系统架构设计
- 复杂Bug诊断
- 性能优化方案
- 安全审计
- 技术选型决策

❌ **不应该用:**
- 简单CRUD代码生成
- 文档格式化
- 常规测试编写
- 代码格式调整
- 批量文本处理

### 2. 提示词优化

**在 OpenCode 中使用特定模型:**

```markdown
# 编码任务 - 明确指定 DeepSeek
[DeepSeek] 实现用户认证模块，包含:
- JWT token 生成和验证
- 密码哈希和验证
- 登录/登出接口

# 文档任务 - 明确指定 Kimi
[Kimi] 分析以下需求文档，生成详细的任务拆解:
[需求文档内容]

# 测试任务 - 明确指定 GLM
[GLM] 为认证模块生成完整的测试套件
```

## 🔧 故障排除

### 问题1: Claude Code 配额用完

**症状**: "Rate limit exceeded"

**解决方案**:
```bash
# 1. 立即切换到 DeepSeek
opencode --model deepseek-coder

# 2. 等待 5 小时后继续使用 Claude Code
# 或使用本地 OpenClaw 处理紧急任务
```

### 问题2: Agent 状态不同步

**症状**: Obsidian 中看不到最新状态

**解决方案**:
```bash
# 手动触发同步
cd /mnt/d/ObsidianVault
git pull

# 或检查 Git Hook
ls -la .git/hooks/post-commit
```

### 问题3: 模型选择不当

**症状**: 代码质量差或 Token 消耗过快

**解决方案**:
```bash
# 查看最近使用的模型
tail -50 ~/.opencode/history.log

# 调整路由规则
vim opencode.json
# 修改 router.rules 部分
```
