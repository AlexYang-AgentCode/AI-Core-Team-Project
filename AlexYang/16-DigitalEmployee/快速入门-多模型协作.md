## 📋 前置条件

- ✅ 已安装 WSL
- ✅ 已安装 Obsidian
- ✅ 已配置 Claude Code (100刀套餐)
- ✅ 已安装 OpenCode (绑定 GLM/Kimi/DeepSeek)
- ✅ 已配置 OpenClaw 本地模型 (可选)

## ⚡ 快速开始 (3步)

### 第1步: 安装 Git Hooks

```bash
cd /mnt/d/ObsidianVault/10-Projects/16-DigitalEmployee

# 安装 post-commit hook
cp Git-Hooks/post-commit-hook.sh .git/hooks/post-commit
chmod +x .git/hooks/post-commit

# 验证
ls -la .git/hooks/post-commit
```

### 第2步: 初始化 Token 统计

```bash
# 初始化 token 统计文件
./Git-Hooks/token-budget-manager.sh init

# 查看当前状态
./Git-Hooks/token-budget-manager.sh stats
```

### 第3步: 开始使用

```bash
# 启动 OpenCode (默认 DeepSeek)
opencode

# 第一个命令: 查看项目状态
> 分析当前项目的进度，生成任务清单
```

## 🎯 日常使用流程

### 早上 (9:00)

```bash
# 1. 启动 OpenCode
opencode

# 2. 用 Kimi 分析项目状态 (便宜、长上下文)
> [Kimi] 分析所有子项目的状态，生成今日待办清单

# 3. 查看 Agent Dashboard
# 在 Obsidian 中打开: 10-Projects/16-DigitalEmployee/Agent-Dashboard.md
```

### 编码工作 (10:00-12:00, 14:00-16:00)

```bash
# 1. 使用 DeepSeek 编码 (主力)
> [DeepSeek] 实现用户认证模块

# 2. 遇到复杂问题? 切换到 Claude Code
claude-code
> 分析认证流程的性能瓶颈

# 3. 获得 Claude 建议后，回到 OpenCode 实现
opencode
> [DeepSeek] 按照刚才的优化方案重构认证代码
```

### 测试与文档 (16:00-18:00)

```bash
# 1. 生成测试 (GLM)
opencode --model glm-4
> 为认证模块生成完整的测试套件

# 2. 生成文档 (Kimi)
opencode --model kimi-k2
> 为认证模块生成 API 文档和使用指南

# 3. 下班前总结
> [Kimi] 总结今日工作进度，生成日报
```

## 💡 模型选择速查表

| 任务类型 | 推荐模型 | 命令 | 原因 |
|---------|---------|------|------|
| **代码生成** | DeepSeek | `opencode` 或 `opencode --model deepseek-coder` | 编码强、便宜、无限制 |
| **文档分析** | Kimi | `opencode --model kimi-k2` | 长上下文、中文好 |
| **测试生成** | GLM | `opencode --model glm-4` | 测试质量高、速度快 |
| **架构设计** | Claude Code | `claude-code` | 推理强、质量高 |
| **代码审查** | Claude Code (关键) / DeepSeek (常规) | `claude-code` 或 `opencode` | 关键代码用 Claude，常规用 DeepSeek |
| **敏感数据** | OpenClaw (本地) | `openclaw` | 无 API 调用、隐私安全 |

## 🔧 使用 Agent 编排器

### 基本用法

```bash
# 架构设计 (自动使用 Claude Code)
./Git-Hooks/agent-orchestrator.sh design requirement.md

# 代码生成 (自动使用 DeepSeek)
./Git-Hooks/agent-orchestrator.sh code architecture.md --project 16.1-AndroidToHarmonyOS

# 测试生成 (自动使用 GLM)
./Git-Hooks/agent-orchestrator.sh test app/auth.py

# 文档生成 (自动使用 Kimi)
./Git-Hooks/agent-orchestrator.sh doc app/auth.py
```

### 高级选项

```bash
# 交互模式 (等待用户输入)
./Git-Hooks/agent-orchestrator.sh code design.md --interactive

# Dry-run (只显示命令，不执行)
./Git-Hooks/agent-orchestrator.sh test app.py --dry-run

# 强制指定模型
./Git-Hooks/agent-orchestrator.sh code design.md --model kimi-k2
```

## 🚨 常见问题

### Q1: Claude Code 配额用完了怎么办?

```bash
# 方案1: 立即切换到 DeepSeek
opencode
> [DeepSeek] 继续刚才的任务

# 方案2: 使用本地 OpenClaw
openclaw
> 处理这个编码任务

# 方案3: 等待 5 小时后继续
```

### Q2: 如何知道该用哪个模型?

```bash
# 方法1: 让编排器建议
./Git-Hooks/token-budget-manager.sh suggest <task-type>

# 方法2: 查看 Dashboard
# 打开 Agent-Dashboard.md，查看"模型使用分布"

# 方法3: Git Hook 会自动建议
# 每次提交后，post-commit hook 会分析并建议最佳模型
```

### Q3: Agent 卡住不动了怎么办?

```bash
# 1. 检查状态文件
cat AGENT-STATUS.md

# 2. 查看是否在等待输入
grep "等待用户输入" AGENT-STATUS.md

# 3. 如果确实卡住，手动更新状态
sed -i "s/status: running/status: error/" AGENT-STATUS.md

# 4. 重新启动任务
./Git-Hooks/agent-orchestrator.sh <task> <input> --interactive
```

### Q4: 如何切换到不同的模型?

```bash
# OpenCode 中切换
opencode --model deepseek-coder  # DeepSeek
opencode --model kimi-k2         # Kimi
opencode --model glm-4           # GLM

# Claude Code
claude-code

# 本地模型
openclaw
```

## 📈 性能优化技巧

### 技巧1: 批量处理

```bash
# ❌ 不好 - 逐个处理
for file in *.py; do
    opencode --model deepseek-coder "优化 $file"
done

# ✅ 好 - 批量处理 (减少会话开销)
opencode --model deepseek-coder "优化以下所有文件: $(ls *.py)"
```

### 技巧2: 使用本地模型处理敏感数据

```bash
# ❌ 不好 - 发送敏感数据到 API
opencode "处理这个包含密码的配置文件..."

# ✅ 好 - 使用本地模型
openclaw "处理这个包含密码的配置文件..."
```

### 技巧3: Claude Code 预算优化

```bash
# ❌ 不好 - 用 Claude 做简单任务
claude-code "格式化这段代码"

# ✅ 好 - 用 DeepSeek 做简单任务，Claude 做复杂决策
opencode "格式化这段代码"  # DeepSeek
claude-code "分析这个架构设计的性能影响"  # Claude
```

### 技巧4: 组合使用

```bash
# 先用 Kimi 理解需求
opencode --model kimi-k2 "分析需求文档，生成任务清单"

# 再用 DeepSeek 实现
opencode --model deepseek-coder "实现第一个任务"

# 然后用 GLM 测试
opencode --model glm-4 "为刚生成的代码编写测试"

# 最后用 Claude 审查关键部分
claude-code "审查核心算法的实现"
```

## 🎓 进阶用法

### 自定义路由规则

编辑 `opencode.json`:

```json
{
  "router": {
    "rules": [
      {
        "pattern": "优化|性能|performance",
        "provider": "claude",
        "reason": "性能优化需要深度推理"
      },
      {
        "pattern": "批量|batch|大量",
        "provider": "deepseek",
        "reason": "批量任务用廉价模型"
      }
    ]
  }
}
```

### 创建项目特定的状态文件

```bash
# 为特定项目创建状态文件
cp 90-Templates/AGENT-STATUS-TEMPLATE.md 16.1-AndroidToHarmonyOS/AGENT-STATUS.md

# 编辑状态文件
vim 16.1-AndroidToHarmonyOS/AGENT-STATUS.md
```

### 设置预算提醒

```bash
# Claude Code 使用超过 70% 时提醒
./Git-Hooks/token-budget-manager.sh alert 70

# 添加到 crontab (每小时检查)
# (crontab -e)
# 0 */1 * * * /mnt/d/ObsidianVault/10-Projects/16-DigitalEmployee/Git-Hooks/token-budget-manager.sh check-alert 70
```
