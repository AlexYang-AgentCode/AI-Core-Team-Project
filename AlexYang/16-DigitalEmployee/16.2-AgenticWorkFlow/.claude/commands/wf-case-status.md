## Case Status Dashboard

查看所有案例的当前状态。三个视图: 流水线视图、阻塞视图、模块负载视图。

参数 $ARGUMENTS 可选:
- 留空: 输出全部三个视图
- 数字 (如 `122`): 该案例的详细状态
- `--blocked`: 只看阻塞视图
- `--load`: 只看模块负载视图

### 执行步骤

1. 扫描 `E:/10.Project/16.1-AndroidToHarmonyOSDemo/16.1.1*` 所有案例目录 (120~142)

2. 对每个案例判定**当前阶段**和**阻塞点**:

```
判定逻辑:
  无 analysis/      → 阶段=TODO,        阻塞=16.1 (未分析)
  有 analysis 无 mapping → 阶段=ANALYZED, 阻塞=16.3 (待设计)
  有 mapping:
    有 Decision C 且 PENDING_APPROVAL → 阶段=WAIT_APPROVAL, 阻塞=Human(P/A)
    有 BRIDGE_EXISTING 但 Bridge 未交付 → 阶段=BRIDGING, 阻塞=16.4.x (具体哪个)
    Bridge 已交付但无测试 → 阶段=WAIT_TEST, 阻塞=16.6
  有 e2e/:
    最新 attempt status:
      DELIVERABLE_INCOMPLETE → 阶段=WAIT_TEST, 阻塞=16.6 (交付不完整)
      FAIL → 阶段=FIXING, 阻塞=具体 Agent (读 error_history)
      PASS → 阶段=DONE, 阻塞=无
  依赖未满足 → 阶段=DEP_BLOCKED, 阻塞=前置案例
```

3. 输出三个视图
