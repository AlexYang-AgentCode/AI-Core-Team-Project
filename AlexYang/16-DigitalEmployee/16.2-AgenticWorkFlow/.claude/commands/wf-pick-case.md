## Pick Next Case

从 16.1 案例队列中选取下一个要处理的案例。

参数 $ARGUMENTS 可选:
- 留空: 自动选取下一个（按序号 + 依赖就绪）
- 数字: 指定案例编号（如 `120`）
- `--list`: 列出所有可选案例

### 执行步骤

1. 扫描 `E:/10.Project/16.1-AndroidToHarmonyOSDemo/16.1.1*` 所有案例目录

2. 读取每个案例的状态:
   - 已有 `e2e/` 且最新 attempt 为 PASS → 已完成，跳过
   - 已有 `e2e/` 且状态为 BLOCKED → 已阻塞，跳过（除非人工解除）
   - 已有 `analysis/` 但未完成 → 进行中
   - 无 `analysis/` → 待开始

3. 检查依赖（参照案例执行序列表）:
   - 案例 124 依赖 121 → 121 未 PASS 则 124 不可选
   - 案例 127 依赖 125 → 125 未 PASS 则 127 不可选
   - 无依赖的案例随时可选

4. 检查 `16.2-AgenticWorkFlow/proposals/` 是否有 PENDING_APPROVAL 的 NMP:
   - 依赖该新模块的案例标记为 WAITING

5. 选取规则（自动模式）:
   - 优先: 已在进行中但未完成的案例（继续推进）
   - 次优: 依赖已满足、序号最小的待开始案例
   - 跳过: BLOCKED、WAITING、依赖未满足的案例
