## 🎯 立即使用 (3步)

### 第1步: 在 VSCode 中打开项目

```bash
# 在 WSL 中运行
code /mnt/d/ObsidianVault
```

### 第2步: 打开终端

```
按 Ctrl + ` (反引号)
```

### 第3步: 启动 OpenCode

```bash
# 进入项目目录
cd 10-Projects/16-DigitalEmployee

# 启动 OpenCode (自动记录日志)
oc
```

**现在你可以:**
- ✅ 用鼠标滚轮自由滚动查看历史输出
- ✅ 按 Ctrl+F 搜索输出内容

## 🔍 查看历史输出

### 方法1: 在 VSCode 终端中滚动

```
启动 OpenCode 后:
1. 直接用鼠标滚轮滚动 ⬆️⬇️
2. 或用键盘: Shift + PgUp/PgDn
3. 搜索: Ctrl + F
```

### 方法2: 查看日志文件

```bash
# 查看最新日志
ocl

# 或用日志管理器
opencode-logs view

# 搜索历史
opencode-logs search "关键词"

# 模糊搜索
opencode-logs fuzzy
```

## 💡 进阶: 安装 tmux (可选)

如果想要更强大的功能（分屏、会话持久化），运行：

```bash
cd /mnt/d/ObsidianVault/10-Projects/16-DigitalEmployee
./Git-Hooks/install-terminal-tools.sh
```

按提示安装 tmux，然后:

```bash
# 方法1: 使用快捷脚本
~/bin/tmux-opencode

# 方法2: 手动创建会话
tmux new -s opencode
oc
```

**tmux 优势:**
- ✅ 分屏显示 (同时看OpenCode和日志)
- ✅ 会话持久化 (关闭终端不会丢失)
- ✅ 复制模式搜索 (类似vim)

## 🚨 常见问题

### Q1: 命令找不到?

```bash
# 重载配置
source ~/.bashrc

# 如果还不行，检查别名
alias | grep oc
```

### Q2: 日志文件在哪?

```bash
# 查看日志目录
ls ~/opencode-logs/

# 查看统计
opencode-logs stats
```

### Q3: 如何搜索历史输出?

```bash
# 方法1: 在 VSCode 终端中
Ctrl + F

# 方法2: 搜索日志文件
opencode-logs search "关键词"

# 方法3: 模糊搜索
opencode-logs fuzzy
```
