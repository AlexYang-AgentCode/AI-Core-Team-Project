## 💡 组合方案 (推荐)

### 方案A: tmux + tee (日常开发)

```bash
# 1. 在 tmux 中启动
tmux new -s opencode

# 2. 启动 OpenCode 并记录日志
opencode 2>&1 | tee -a ~/opencode-logs/$(date +%Y%m%d).log

# 优势:
# ✅ tmux 提供滚动和分屏
# ✅ tee 提供日志归档
# ✅ 会话持久化
```

### 方案B: VSCode终端 + 自动日志 (简单高效)

```bash
# 1. 在 VSCode 终端中添加别名
# ~/.bashrc
alias oc='opencode 2>&1 | tee -a ~/opencode-logs/$(date +%Y%m%d).log'

# 2. 使用
oc

# 优势:
# ✅ VSCode 图形界面滚动
# ✅ Ctrl+F 搜索
# ✅ 自动保存日志
```

### 方案C: 多终端方案 (最灵活)

```
日常编码 → VSCode终端 (快速滚动、搜索)
重要会话 → tmux (会话持久化、分屏监控)
历史归档 → 日志文件 (长期保存、搜索)
```

## 🚀 快速开始指南

### 如果你选择 tmux (推荐给高级用户)

```bash
# 1. 安装
sudo apt install tmux -y

# 2. 创建配置
cat > ~/.tmux.conf << 'EOF'
set -g mouse on
set -g history-limit 50000
set -g prefix C-a
unbind C-b
bind | split-window -h
bind - split-window -v
EOF

# 3. 启动
tmux new -s opencode
cd /mnt/d/ObsidianVault/10-Projects/16-DigitalEmployee
opencode

# 4. 使用鼠标滚动查看历史!
# 5. 按 Ctrl+a d 后台运行，tmux attach -t opencode 恢复
```

**学习时间**: 30分钟  
**配置时间**: 5分钟  
**适用**: 长期使用

---

### 如果你选择 VSCode终端 (推荐给新手)

```bash
# 1. 打开 VSCode
code /mnt/d/ObsidianVault

# 2. 打开终端
Ctrl + `

# 3. 启动 OpenCode
cd 10-Projects/16-DigitalEmployee
opencode

# 4. 直接用鼠标滚轮滚动查看历史!
# 5. 按 Ctrl+F 搜索
```

**学习时间**: 2分钟  
**配置时间**: 0分钟  
**适用**: 临时使用、简单场景

---

### 如果你选择日志方案 (推荐给需要归档的用户)

```bash
# 1. 创建日志目录
mkdir -p ~/opencode-logs

# 2. 添加别名
echo "alias oc='opencode 2>&1 | tee -a ~/opencode-logs/\$(date +%Y%m%d).log'" >> ~/.bashrc
source ~/.bashrc

# 3. 使用
oc

# 4. 查看日志
less ~/opencode-logs/$(date +%Y%m%d).log

# 5. 搜索历史
rg "关键词" ~/opencode-logs/
```

**学习时间**: 5分钟  
**配置时间**: 2分钟  
**适用: 需要保存历史

## 📋 详细方案说明

### 1. tmux - 终端增强神器

**核心功能:**
- 🖱️ 鼠标滚动查看历史 (解决你的痛点)
- 📊 分屏显示 (同时查看OpenCode和日志)
- 💾 会话持久化 (关闭终端不会丢失)
- 🔍 复制模式搜索 (类似vim)

**适合人群:**
- 经常使用命令行
- 需要分屏和多窗口
- 追求效率和自动化
- 愿意花时间学习

**详细指南**: [[tmux使用指南]]

## 💬 常见问题

### Q1: 能不能同时使用多个方案?

✅ 可以! 推荐组合:
- 日常: VSCode终端 (简单快速)
- 重要会话: tmux (持久化、分屏)
- 历史查看: 日志文件 (搜索、归档)

### Q2: 哪个方案性能最好?

tmux ≈ 日志文件 > VSCode终端

但对于OpenCode这种IO密集型任务，差异不明显。

### Q3: 我该花多少时间学习?

- VSCode终端: 5分钟 (立即使用)
- 日志文件: 10分钟 (今天学会)
- tmux: 2小时 (本周学习)

建议: 先用VSCode终端解决眼前问题，再逐步学习tmux。
