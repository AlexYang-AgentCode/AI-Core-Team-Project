# P7885 OpenHarmony 编译环境搭建指南

## 1. 环境要求

### 1.1 硬件要求
| 资源 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 8核 x86_64 | 16核+ |
| 内存 | 16GB | 32GB+ |
| 磁盘 | 200GB SSD | 500GB+ SSD |
| 网络 | 10Mbps | 100Mbps+ |

### 1.2 操作系统
- **推荐**: Ubuntu 22.04 LTS (x86_64)
- **可选**: Ubuntu 20.04 LTS / Debian 11
- **不支持**: Windows (WSL2可尝试但不推荐)

## 2. 基础环境安装

### 2.1 系统更新
```bash
sudo apt-get update
sudo apt-get upgrade -y
```

### 2.2 安装依赖包
```bash
sudo apt-get install -y \
    binutils git git-lfs gnupg flex bison gperf build-essential zip \
    curl zlib1g-dev gcc-multilib g++-multilib libc6-dev-i386 \
    lib32ncurses5-dev x11proto-core-dev libx11-dev lib32z1-dev ccache \
    libgl1-mesa-dev libxml2-utils xsltproc unzip m4 bc gnutls-bin \
    python3.10 python3-pip python3-venv \
    ruby genext2fs device-tree-compiler \
    make libffi-dev e2fsprogs pkg-config perl \
    openssl libssl-dev libelf-dev libdwarf-dev u-boot-tools \
    mtd-utils cpio doxygen liblz4-tool openjdk-17-jdk \
    default-jdk llvm libre2-9 libc6 libncurses5 \
    libtinfo5 libstdc++6 gcc-arm-linux-gnueabi
```

### 2.3 配置Python
```bash
# 设置默认Python版本
sudo update-alternatives --install /usr/bin/python python /usr/bin/python3.10 1

# 升级pip
pip3 install --upgrade pip

# 安装Python依赖
pip3 install --user build kconfiglib pycryptodome pyyaml json2html
```

### 2.4 安装repo工具
```bash
curl -s https://gitee.com/oschina/repo/raw/fork_flow/repo-py3 > /tmp/repo
sudo cp /tmp/repo /usr/local/bin/repo
sudo chmod a+x /usr/local/bin/repo
pip3 install --user requests
```

### 2.5 安装Node.js (14.x)
```bash
curl -fsSL https://deb.nodesource.com/setup_14.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 2.6 安装GN和Ninja
```bash
# 下载预编译版本
wget https://repo.huaweicloud.com/harmonyos/compiler/gn/1717/linux/gn-linux-x86-1717.tar.gz
wget https://repo.huaweicloud.com/harmonyos/compiler/ninja/1.10.1/linux/ninja-linux-x86-1.10.1.tar.gz

# 解压安装
tar -xzf gn-linux-x86-1717.tar.gz
tar -xzf ninja-linux-x86-1.10.1.tar.gz

sudo cp gn-linux-x86-1717/gn /usr/local/bin/
sudo cp ninja-linux-x86-1.10.1/ninja /usr/local/bin/

# 验证
gn --version  # 1717
ninja --version  # 1.10.1
```

### 2.7 安装Clang编译器
```bash
# OpenHarmony 4.1 推荐 LLVM 15.0.4
wget https://repo.huaweicloud.com/harmonyos/compiler/clang/15.0.4/linux/clang-15.0.4-linux-x86_64.tar.gz
tar -xzf clang-15.0.4-linux-x86_64.tar.gz

# 添加到环境变量
echo 'export PATH=$HOME/clang-15.0.4/bin:$PATH' >> ~/.bashrc
echo 'export CC=clang' >> ~/.bashrc
echo 'export CXX=clang++' >> ~/.bashrc
source ~/.bashrc

# 验证
clang --version
```

## 3. 下载OpenHarmony源码

### 3.1 创建源码目录
```bash
mkdir -p ~/ohos-p7885 && cd ~/ohos-p7885
```

### 3.2 配置Git
```bash
git config --global user.name "your-name"
git config --global user.email "your-email@example.com"
git config --global credential.helper store
```

### 3.3 初始化Repo仓库
```bash
# OpenHarmony 4.1 Release 分支
repo init -u https://gitee.com/openharmony/manifest.git -b OpenHarmony-4.1-Release --no-repo-verify

# 或主干分支
# repo init -u https://gitee.com/openharmony/manifest.git -b master --no-repo-verify
```

### 3.4 同步源码
```bash
# 全量同步（耗时较长，约30-60分钟）
repo sync -c --no-tags -j4

# 或只下载P7885相关代码（推荐）
repo sync -c --no-tags -j4 \
    build/prebuilts_download.sh \
    foundation/arkui/ace_engine \
    device/soc/allwinner \
    device/board/hihope \
    vendor/hihope
```

### 3.5 下载预编译工具链
```bash
cd ~/ohos-p7885
./build/prebuilts_download.sh

# 或只下载必要组件
./build/prebuilts_download.sh --skip-ssl --no-check-certificate \
    --skip-nodejs --skip-gn --skip-ninja --skip-clang
```

## 4. 编译P7885镜像

### 4.1 编译前准备
```bash
cd ~/ohos-p7885

# 加载环境变量
source build/envsetup.sh

# 选择编译目标
# lunch 后会显示可选目标，选择 hihope_p7885
lunch

# 或直接指定
lunch hihope_p7885-userdebug
```

### 4.2 执行编译
```bash
# 完整编译（耗时约30-60分钟，视硬件配置）
./build.sh --product hihope_p7885 --ccache

# 编译选项说明
# --product: 产品名称
# --ccache: 启用编译缓存
# --jobs/-j: 并行编译任务数
# --verbose: 显示详细输出
```

### 4.3 编译输出
```bash
# 编译成功后，镜像位于:
out/hihope_p7885/packages/phone/images/

# 主要文件
├── boot.img          # 启动镜像
├── system.img        # 系统镜像
├── vendor.img        # 厂商镜像
├── userdata.img      # 用户数据镜像
├── ramdisk.img       # 内存盘镜像
└── updater.img       # 升级镜像
```

```bash
  ]
```

```bash
{
  "subsystem": "ai",

    {"component": "ai_engine", "features": []}
```

```bash
make -C kernel/linux/linux-5.15 ARCH=arm64 menuconfig
```

## 6. 编译加速技巧

### 6.1 使用ccache
```bash
# 启用ccache（默认已启用）
export CCACHE_DIR=~/.ccache
export CCACHE_MAXSIZE=50G

# 查看缓存统计
ccache -s
```

### 6.2 分布式编译（ccache + distcc）
```bash
# 安装distcc
sudo apt-get install distcc distcc-pump

# 配置编译服务器列表
export DISTCC_HOSTS="localhost/16,cpp,lzo 192.168.1.100/16,cpp,lzo"

# 使用distcc编译
./build.sh --product hihope_p7885 --jobs 32
```

### 6.3 增量编译
```bash
# 只编译修改的部分
./build.sh --product hihope_p7885 --ccache

# 清理特定模块后重新编译
rm -rf out/hihope_p7885/your_module
./build.sh --product hihope_p7885
```

## 7. 常见问题

### Q1: 编译报错 "out of memory"
```bash
# 解决方案：减少并行任务数
./build.sh --product hihope_p7885 --jobs 4

# 或增加swap空间
sudo fallocate -l 16G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### Q2: 依赖下载失败
```bash
# 使用国内镜像
export REPO_URL='https://gitee.com/oschina/repo/raw/fork_flow/repo-py3'
repo init -u https://gitee.com/openharmony/manifest.git -b OpenHarmony-4.1-Release

# 手动下载prebuilts
./build/prebuilts_download.sh --skip-ssl --no-check-certificate
```

### Q3: 编译过程中断
```bash
# 继续编译
./build.sh --product hihope_p7885 --ccache

# 或清理后重编
rm -rf out/hihope_p7885
./build.sh --product hihope_p7885
```

## 8. 自动化编译脚本

创建 `auto_build.sh`:
```bash
#!/bin/bash
set -e

# 配置
PRODUCT="hihope_p7885"
BUILD_DIR="$HOME/ohos-p7885"
LOG_FILE="/var/log/ohos-build-$(date +%Y%m%d-%H%M%S).log"

# 记录开始时间
START_TIME=$(date +%s)

echo "=== OpenHarmony Build Started at $(date) ===" | tee -a $LOG_FILE

# 进入源码目录
cd $BUILD_DIR

# 加载环境
source build/envsetup.sh
lunch ${PRODUCT}-userdebug

# 执行编译
./build.sh --product $PRODUCT --ccache 2>&1 | tee -a $LOG_FILE

# 检查编译结果
if [ -f "out/${PRODUCT}/packages/phone/images/boot.img" ]; then
    END_TIME=$(date +%s)
    DURATION=$((END_TIME - START_TIME))
    echo "=== Build Success! Duration: ${DURATION}s ===" | tee -a $LOG_FILE
    
    # 复制镜像到输出目录
    mkdir -p /opt/ohos-images/$(date +%Y%m%d)
    cp out/${PRODUCT}/packages/phone/images/*.img /opt/ohos-images/$(date +%Y%m%d)/
    exit 0
else
    echo "=== Build Failed! Check log: $LOG_FILE ===" | tee -a $LOG_FILE
    exit 1
fi
```

## 9. 验证编译环境

运行检查脚本：
```bash
#!/bin/bash
echo "=== OpenHarmony Build Environment Check ==="

# 检查系统版本
echo "OS Version: $(lsb_release -d | cut -f2)"

# 检查Python
echo "Python: $(python3 --version)"

# 检查Node.js
echo "Node.js: $(node --version)"

# 检查Git
echo "Git: $(git --version)"

# 检查repo
echo "Repo: $(repo --version | head -1)"

# 检查磁盘空间
DF=$(df -h . | tail -1 | awk '{print $4}')
echo "Available Disk: $DF"

# 检查内存
MEM=$(free -h | grep Mem | awk '{print $7}')
echo "Available Memory: $MEM"

# 检查编译工具
echo "GN: $(gn --version 2>/dev/null || echo 'Not installed')"
echo "Ninja: $(ninja --version 2>/dev/null || echo 'Not installed')"
echo "Clang: $(clang --version 2>/dev/null | head -1 || echo 'Not installed')"

echo "=== Check Complete ==="
```
